# ESBot - Software Requirements Specification

**Version:** 0.1.0
**Date:** 2026-03-28

## 1. Introduction

This document provides a detailed specification of the requirements for the ESBot project. ESBot is an educational chatbot designed to assist students with course-related questions and practice quizzes. This specification covers the functional requirements (Use Cases) and non-functional requirements (Quality Model).

## 2. System Overview

ESBot will feature a 3-layered architecture:

- **Presentation Layer (Frontend):** A Streamlit web interface where students interact with the bot.
- **Application Layer (Backend):** A FastAPI application that processes requests, handles business logic, and communicates with the AI and data layers.
- **Data Layer (Database):** A PostgreSQL database, managed in a Docker container, for storing course materials, chat history, and other persistent data.
- **AI Inference:** The Groq API will be used for all language model processing to generate answers and quizzes.

## 3. Functional Requirements (Use Cases)

This section details the primary functionalities of the system from a user's perspective.

### 3.1. Use Case: Ask Question

|                    |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| :----------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **ID**             | UC-001                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **Name**           | Ask a Course-Related Question                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Actor**          | Student                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Description**    | A student submits a question about their course material through the Streamlit chat interface. The system processes the question, retrieves relevant context from the database, and uses the Groq API to generate a comprehensive explanation.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Trigger**        | The student types a question into the chat input field and submits it.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **Preconditions**  | 1. The student has accessed the ESBot Streamlit application. <br> 2. The backend services (FastAPI, PostgreSQL, Groq API) are operational.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Basic Flow**     | 1. The student enters a question into the chat UI and presses "Send". <br> 2. The Streamlit frontend sends the question to the FastAPI backend via an API call. <br> 3. The backend receives the question and queries the PostgreSQL database to find relevant context (e.g., lecture notes, textbook excerpts). <br> 4. The backend constructs a prompt containing the student's question and the retrieved context. <br> 5. The backend sends the prompt to the Groq API for inference. <br> 6. The Groq API returns a generated answer. <br> 7. The backend saves the question and its answer to the chat history in the PostgreSQL database. <br> 8. The backend returns the answer to the Streamlit frontend. <br> 9. The frontend displays the answer to the student in the chat interface. |
| **Postconditions** | 1. The student sees the answer to their question. <br> 2. The conversation (question and answer) is saved in the student's chat history.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |

## 4. Non-Functional Requirements (Quality Model)

This section is based on the ISO/IEC 25010 quality model and defines the quality attributes for the system.

### 4.1. Performance Efficiency

|                       |                                                                                                                                                      |
| :-------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                | NFR-001                                                                                                                                              |
| **Quality Attribute** | **Time Behavior**                                                                                                                                    |
| **Description**       | The system must respond to user interactions in a timely manner to ensure a fluid user experience.                                                   |
| **Requirement**       | The end-to-end API response time for the "Ask Question" use case (from student submission to answer display) must be **under 5 seconds** on average. |

### 4.2. Usability

|                       |                                                                                                                                                                                                       |
| :-------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                | NFR-002                                                                                                                                                                                               |
| **Quality Attribute** | **Learnability**                                                                                                                                                                                      |
| **Description**       | The system should be intuitive and easy to use for the target audience (students) without prior training.                                                                                             |
| **Requirement**       | The chat interface must be simple and self-explanatory. A first-time user should be able to successfully ask a question and understand the response without needing a tutorial or help documentation. |

### 4.3. Reliability

|                       |                                                                                                                                                                                                                                                                 |
| :-------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ID**                | NFR-003                                                                                                                                                                                                                                                         |
| **Quality Attribute** | **Recoverability**                                                                                                                                                                                                                                              |
| **Description**       | The system must be able to recover from failures (e.g., dropped connection, browser crash) without data loss.                                                                                                                                                   |
| **Requirement**       | All chat history, including every question asked by the student and every answer generated by the system, must be persistently stored in the PostgreSQL database. If a user's session is interrupted, the full chat history must be restored upon their return. |
